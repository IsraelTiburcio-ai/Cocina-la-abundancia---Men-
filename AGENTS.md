# AGENTS.md — Cocina La Abundancia

Static landing page for a Mexican restaurant with a WhatsApp order flow. No build step, no package manager, no tests, no linter.

## Stack
- HTML + CSS + vanilla JS (no framework, no bundler)
- JS split into 11 ES modules under `js/` (entrypoint: `js/main.js`); no build step, loaded directly via `<script type="module">`
- Leaflet 1.9.4 via CDN for the location picker map (loaded as classic defer script before the module)
- Google Maps embed for the "Conócenos" section
- Menú images served from Cloudinary with `f_auto,q_auto` URL transforms (WebP/AVIF/JPEG auto-negotiated)
- Hero image served as `Portada.webp` with `Portada.png` PNG fallback via CSS `image-set()`
- JSON-LD (Restaurant, Menu, Organization, WebSite) generated by `scripts/build-jsonld.mjs` and inlined in `<head>`
- Deploy: GitHub Pages via `.github/workflows/deploy.yml` on push to `main`

## File ownership
- `index.html` — markup, JSON-LD, all meta tags (title, description, OG, Twitter, canonical, hreflang, robots), WhatsApp deep links
- `styles.css` — all styles, cache-busted via `?v=...` query string
- `js/` — 11 ES modules:
  - `main.js` — entrypoint, wires events, owns module init order
  - `dom.js` — all DOM references as a single `el` object
  - `state.js` — global state + `localStorage` persistence (key `cla_order_state_v3`, 24h TTL)
  - `availability.js` — pure functions: schedule parsing, day-of-week logic
  - `ui-helpers.js` — currency, focus trap, scroll lock, toast/announce
  - `cart.js` — add/remove/qty, totals, Haversine distance, shipping
  - `menu-render.js` — chips, product grid, landing menus, cart, order bar, config sheet
  - `map-picker.js` — Leaflet init, geolocation, map click handler
  - `checkout.js` — form sync, validation
  - `whatsapp.js` — message builder, `ORDER_PHONE`, popup-blocker fallback, URL length check
  - `navigation.js` — IntersectionObserver reveal, mobile nav
- `menu-data.js` — exposes `window.MENU_DATA` AND `export { MENU_DATA }`: products, categories, pricing, business coords, schedules
- `scripts/build-jsonld.mjs` — regenerates the JSON-LD in `index.html` from `menu-data.js`. Re-run whenever menu/business info changes
- `Portada.png` + `Portada.webp` — hero background (CSS picks WebP)
- `Menu-V2/` — also not referenced in HTML; exists only to satisfy the `validate-assets` workflow job that walks this directory. Keep the folder, but its contents are free to remove/replace
- `robots.txt` + `sitemap.xml` — at repo root, allow all + point to sitemap
- `.nojekyll` — required for GitHub Pages to serve files starting with `_`. Keep it
- `AGENTS.md` — this file

## Cache busting
`index.html` loads `styles.css?v=20260708-2` and `js/main.js?v=20260708-2`. Bump both strings on any change to `styles.css`, `script.js` (deleted, replaced by `js/*`), or `js/main.js`. Cache-busting the module path is also important because browsers cache the module graph.

## Deploy
Push to `main` triggers `.github/workflows/deploy.yml`:
1. `deploy` job publishes the repo root to GitHub Pages. The site URL output is `https://israeltiburcio-ai.github.io/Cocina-la-abundancia---Men-/`.
2. `validate-assets` job (runs after deploy) does an HTTP check on:
   - `index.html`, `Portada.png`, `Portada.webp`, `styles.css`, `menu-data.js`, `js/main.js`
   - Every file under the directory whose name starts with `menu` and contains `V2` (case-insensitive) — current match: `Menu-V2/`. If you rename/move that folder, the validator breaks.

There is no staging environment. Every push to `main` is live.

## Local preview
No dev server. Open `index.html` directly in a browser, or `python3 -m http.server` from the repo root. The site needs network access for Leaflet CDN, Google Maps embed, and Cloudinary images.

## Conventions / gotchas
- **Order state** is persisted in `localStorage` under key `cla_order_state_v3` (was `_v2`, now schema-versioned with `version: 3` field inside the payload) with a 24h TTL (`STORAGE_KEY`, `STORAGE_TTL_MS` in `js/state.js`).
- **Prices** are integers in MXN. Shipping is `$10/km` from `MENU_DATA.businessLocation`; packaging is `$10/item`; pickup has a fixed charge (currently `$0`).
- **Category availability** uses `availability.mode = 'schedule'` with `days` as JS day-of-week numbers (0=Sun … 6=Sat). Pozole is `[5,6]`, Antojitos is `[5,6]`. Products also have `availability` and the cart respects it via `isProductAvailableNow` in `js/cart.js`.
- **Product modifiers** are defined inline in `menu-data.js` (`modifiers[]` with `type: 'multi' | 'single'`, `required`, and `priceDelta` per option).
- **WhatsApp message** has a 2000-char URL length check in `js/whatsapp.js` before `window.open`.
- **Title tag** must stay under 60 chars (Google truncates). Current: 45 chars.
- **Meta description** must stay under 160 chars. Current: 129 chars.
- **JSON-LD** must be regenerated via `scripts/build-jsonld.mjs` whenever `menu-data.js` or business info changes. The script's output is what's inlined in `index.html`.
- **`.gitignore`** only ignores `.DS_Store`. The `.agents/` and `skills-lock.json` files are opencode session artifacts, intentionally untracked. `Portada.webp` is tracked (was committed during Fase 2).
